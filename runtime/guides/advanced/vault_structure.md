# Vault Structure

**Level:** Advanced

---

## Top-level layout

```
ai_framework/              ← project root (rename freely)
├── runtime_core/          ← all runtime files
│   ├── conf/
│   ├── exec/
│   ├── lib/
│   ├── logs/
│   ├── modules/
│   ├── scripts/
│   └── state.json
└── ai-vault/              ← Obsidian-compatible vault (sibling, not inside runtime)
    ├── .obsidian/         ← Obsidian config (ignored by runtime)
    ├── 000_indexes/
    ├── 001_topics/
    ├── 002_sessions/
    ├── 003_raw/
    ├── 004_prompts/
    ├── 005_tags/
    ├── 006_knowledge/
    ├── 007_guides/
    ├── 008_agents/
    ├── 009_tutorials/
    └── bundles/
```

**Rule:** `ai-vault/` is never inside `runtime_core/`. The runtime path
`vault_root = "../ai-vault"` resolves relative to `runtime_core/`, placing
the vault one level up and beside it.

---

## Vault folder purposes

| Folder | Key | Contents |
|---|---|---|
| `000_indexes/` | `indexes` | Auto-generated index files. Never edit manually. |
| `001_topics/` | `topics` | Topic notes, linked to sessions by keyword |
| `002_sessions/` | `sessions` | Archived session files, organised by month |
| `003_raw/` | `raw` | Raw prompt paste files, organised by month |
| `004_prompts/` | `prompts` | Prompt files with YAML frontmatter |
| `005_tags/` | `tags` | Tag definition files (future use) |
| `006_knowledge/` | `knowledge` | Knowledge packs and reference docs |
| `007_guides/` | `guides` | This guide system |
| `008_agents/` | `agents` | Agent definition files (future use) |
| `009_tutorials/` | `tutorials` | Step-through tutorial packs |
| `bundles/` | `bundles` | Prompt bundles (groups of prompts) |

---

## Session files

Session files are stored in `002_sessions/YYYY-MM/`:

```
002_sessions/
└── 2026-03/
    ├── 2026-03-07-LOG-0001.md
    ├── 2026-03-07-LOG-0002.md
    └── ...
```

Filename format: `YYYY-MM-DD-LOG-NNNN.md`  
The log number matches the `log:` field in the SESSION block header.

---

## Raw files

Raw files are the unprocessed paste from your AI chat. Stored in `003_raw/YYYY-MM/`:

```
003_raw/
└── 2026-03/
    ├── 2026-03-07-RAW-0001.md
    └── ...
```

The cleaner matches raw files to sessions by date and log number.

---

## Prompt files

Each prompt is a Markdown file in `004_prompts/` with YAML frontmatter:

```markdown
---
alias: research_mode
type: workflow
tags: [research, depth]
status: active
created: 2026-03-07
---

# Research Mode

Approach every topic with systematic depth...
```

**Required frontmatter fields:**

| Field | Purpose |
|---|---|
| `alias` | Short name used in SESSION blocks (`* alias`) |
| `type` | `task`, `workflow`, `style`, `format`, or `constraint` |
| `status` | `active`, `draft`, `archived`, or `disabled` |

---

## Bundle files

Bundles group prompts for activation together. Stored in `bundles/`:

```markdown
---
bundle: deep_research
prompts: [research_mode, depth_first, cite_sources]
---
```

Activate a bundle from the prompter with `*bundle_name`.

---

## Index files

The `000_indexes/` folder contains auto-generated files. Do not edit:

| File | Generated by |
|---|---|
| `session_index.md` | cleaner (on each archive run) |
| `prompt_index.json` | manager (on each scan) |
| `tag_index.md` | manager (future) |

---

## Obsidian compatibility

The vault is structured to work with Obsidian:
- All files are Markdown
- YAML frontmatter is standard
- Folder names use numeric prefixes for sort order
- No proprietary formats

You can open `ai-vault/` directly in Obsidian as a vault.
The `000_indexes/` prefix ensures index files appear first.

---

## Moving the vault

To move the vault to a different relative location:

1. Take a snapshot first
2. Move the `ai-vault/` folder
3. Update `vault_root` in `runtime_core/conf/system_config.json`
4. Run the prompter once to verify bootstrap
